이진 탐색 — Binary Search

📌 문제

정렬된 배열 [-1, 0, 3, 5, 9, 12]에서 9의 위치를 찾아라.

💡 핵심 아이디어

  • 배열의 중간값(mid)을 확인한다.
  • mid가 타겟보다 작으면 오른쪽 절반을 탐색한다.
  • mid가 타겟보다 크면 왼쪽 절반을 탐색한다.

⏱ 복잡도

시간: O(log n)공간: O(1)

🏗 아키텍트의 시선

정렬된 데이터에서 탐색할 때 가장 기본이 되는 알고리즘. O(log n)의 강력한 성능을 자랑한다.
