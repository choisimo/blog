awk는 고수준 인터프리터 언어로 하드웨어의 복잡성을 추상화하지만, 내부적으로 NR 변수와 비교/논리 연산자(>, ==, &&)가 
처리되는 과정은 시스템의 메모리 관리 및 CPU의 하드웨어 레벨(비트 연산 및 상태 플래그)과 밀접하게 연관되어 있습니다. 

### 1. NR 변수의 메모리 및 시스템 레벨 처리
awk 프로그램은 입력 스트림을 연속적인 데이터가 아닌 레코드(일반적으로 한 줄) 단위로 처리하며, 각 레코드를 내부 버퍼(Internal buffer)
메모리 공간에 복사합니다. 이때 NR은 현재 처리 중인 레코드 번호를 저장하는 내장 변수로, 
초기값 1에서 시작해 새로운 레코드를 읽을 때마다 1씩 증가합니다.

![NR](images/2026/awk-1/1.png) ![NR2](images/2026/awk-1/2.png)

**메모리 주소와 할당 방식**: awk 변수들은 C나 Java처럼 명시적인 데이터 타입 선언 없이 사용되며, 실행 문맥(Context)에 따라 문자열이나 숫자로 동적 변환됩니다. 내부적으로 이러한 변수들은 **심볼 테이블(Symbol table)**이나 환경(Environment) 구조체 배열에 저장되어 관리됩니다,. 시스템은 런타임 스택(Run-time stack) 메모리에 **활성화 레코드(Activation record 또는 Frame)**를 푸시(Push)하여 프로시저와 로컬 변수들을 관리합니다.

**주소 계산**: 하드웨어 레벨에서 NR과 같은 변수에 접근할 때는 메모리의 절대 주소를 직접 참조하기보다, 레지스터(예: 스택 포인터)에 저장된 기준 주소에 오프셋(Offset)을 더하는 **인덱스 주소 지정 방식(Indexed address mode)**을 사용하여 변수가 위치한 메모리 주소를 계산하고 값을 로드(Load)하거나 스토어(Store)합니다

### 2. 비교 연산자 (예시 : >, ==) 의 하드웨어 비트 연산
awk의 스크립트에서 사용된 비교 연산은 최종적으로 CPU의 **산술 논리 장치(ALU, Arithmetic and Logic Unit)**에서 수행되는 하드웨어 레벨의 비트 연산 및 상태 플래그 확인 과정으로 변환됩니다.

![x : 4 == y : 4](images/2026/awk-1/3.png)
**비교를 위한 내부 뺄셈 연산**: 하드웨어는 두 값 x와 y를 비교할 때 주로 **뺄셈(x - y)**을 내부적으로 수행합니다.
이때 프로세서는 실제 뺄셈 명령어 대신 y의 비트를 반전시킨 후 1을 더하는 2의 보수(Two's complement) 방식(x + ~y + 1)을 사용하여 
가산기(Adder)를 통해 연산합니다. 

![x : 4 == y : 4](images/2026/awk-1/4.png) 
![x : 4 == y : 5](images/2026/awk-1/5.png) 
**상태 레지스터(PSW)와 플래그(Condition Codes)**: 연산 결과값 자체를 저장하기보다, 연산 결과에 따라 프로그램 상태 워드(Program Status Word, PSW) 레지스터의 특정 비트(플래그)들이 세팅됩니다. 주요 플래그로는 **Z(Zero 플래그), N(Negative/Sign 플래그), V(Overflow 플래그), C(Carry 플래그)**가 있습니다,.

![x : 4 == y : 4](images/2026/awk-1/6.png) 
![x : 4 == y : 3](images/2026/awk-1/7.png) 
**== (Equal to) 연산**: x == y인지 판별하기 위해 CPU는 x - y의 결과가 0인지 확인합니다.
결과의 모든 비트가 0일 경우에만 Z(Zero) 플래그가 1로 세팅되며, 하드웨어는 오직 이 Z 플래그의 활성화 여부만으로 동등성을 판단합니다.

![x : 4 > y : 4](images/2026/awk-1/8.png) 
![x : 4 > y : 5](images/2026/awk-1/9.png) 
![x : 4 > y : 3](images/2026/awk-1/10.png) 
**> (Greater than) 연산**: x > y 연산의 평가는 훨씬 복잡한 비트 플래그 조합을 확인합니다. 프로세서는 x - y를 수행한 후, (N == V) & ~Z (Sign 플래그와 Overflow 플래그가 같고, Zero 플래그가 0임)와 같은 논리식을 하드웨어 게이트 레벨에서 계산하여 1비트의 참/거짓 결과(1 또는 0)를 도출합니다.

### 3. 논리 연산자 (&&)의 시스템적 원리
&& 연산자는 두 조건이 모두 참일 때만 참을 반환하는 논리 AND 연산자입니다. 
하드웨어의 비트 논리곱(&) 연산이 각 비트 자리를 1대 1로 비교하여 결과를 내는 것과 달리, 
&& 연산자는 시스템 레벨에서 **단락 평가(Short-circuit evaluation)**라는 최적화된 흐름 제어 아키텍처로 구현됩니다.

![PC](images/2026/awk-1/11.png) 
**단락 평가 아키텍처**: 식 조건1 && 조건2에서 조건1이 거짓(False)으로 판명되면 전체 결과는 이미 거짓이 확정되므로
조건2의 메모리 접근이나 연산은 아예 수행되지 않습니다.

![TRUE](images/2026/awk-1/12.gif)
![FALSE](images/2026/awk-1/13.gif)
**제어 흐름과 점프 명령어(Jump/Branch)**: 컴파일러나 인터프리터가 && 연산자를 중간 코드(Intermediate Code)나 기계어로 번역할 때, 이를 비트 연산 명령어가 아닌 조건부 분기(Conditional Branch) 명령어로 구현합니다. 예를 들어 첫 번째 조건의 평가 결과가 거짓(0)이면, 프로세서의 프로그램 카운터(PC)를 조건2를 건너뛰고 해당 논리 블록의 끝(False 라벨)으로 바로 이동(Jump)시키도록 지시하여 불필요한 레지스터 연산이나 메모리 로드 횟수를 줄입니다.
